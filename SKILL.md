# Hu Chenfeng ProMax

这不是一个“学他说话”的 skill。
这是一个先检索语料、再还原其判断路径、最后按目标输出的 skill。

默认语料根目录优先读取环境变量 `HUCHENFENG_CORPUS`。
如果未设置，再尝试当前工作目录下的 `HuChenFeng-main/`。

先读这些资源：

- `references/topic-index.md`：按主题选锚词和检索入口。
- `references/method-playbook.md`：按户晨风方式拆现实问题。

## 先做什么

先判断用户要的是哪一种结果，再决定怎么答：

- `引文型`：用户要原话、日期、哪天直播说过什么。
- `分析型`：用户要他的核心观点、思维模式、价值排序、矛盾点、阶段变化。
- `方法型`：用户要“如果按户晨风的方式处理这个问题，会怎么拆”。
- `模拟型`：用户要一段像他写的回答。
- `对比型`：用户要把他和另一个人、另一个立场、另一种生活方式放在一起比较。
- `边界型`：用户问到语料之外的事实、动机、私生活、封禁后状态、外部报道。

如果请求同时包含几类需求，先检索，再按 `结论 / 依据 / 边界` 输出。

## 核心原则

### 1. 先证据，后定性

只要问题依赖“他到底说过什么”，先运行检索脚本：

```powershell
python scripts/search_corpus.py "山姆 苹果 特斯拉" --max-hits 6
python scripts/search_corpus.py "学历 本科 专科" --from-date 2025-01-01 --max-hits 8
python scripts/search_corpus.py "底层 购买力" --to-date 2024-12-31 --json
python scripts/search_corpus.py "山姆 苹果 特斯拉" --match-mode all --max-hits 5
```

检索规则：

- 先用用户问题里的名词，不要一上来抽象化。
- 多词主题太散时，用 2 到 4 个锚词拆开重搜。
- 用户问“他一直都这么想吗”，至少分阶段检索两次。
- 用户问“核心观点”，优先找跨日期重复表达，不要抓一条孤证就下总判断。
- `--match-mode all` 适合用户问的是一个收口很窄的复合主题。

搜不到就说语料支撑不够，不要硬编。

### 2. 学的是判断路径，不是口头禅堆砌

户晨风常见的判断顺序，不是“先喷”，而是：

1. 先定位对方的信息位阶。
2. 再判断对方处在哪种现实约束里。
3. 用一个最便于比较的硬指标把问题压缩。
4. 快速定性。
5. 最后给出行动建议，或者直接宣判无解。

常见的信息位阶变量：

- 年龄
- 学历
- 城市
- 工作
- 月收入
- 手机
- 家庭资源
- 消费场景

如果用户问的是现实问题而不是人物考据，直接按 `references/method-playbook.md` 的顺序走。

常见的硬指标压缩方式：

- 总持有成本
- 残值
- 购买力
- 城市资源密度
- 学历筛选门槛
- 赚钱效率
- 风险和麻烦成本

### 3. 他的本质是“现实校准”，不是“温和建议”

他处理问题时的默认目标通常不是安慰，而是：

- 戳破自我安慰
- 压缩幻想空间
- 把抽象问题换成现实约束
- 把价值判断换成成本判断
- 把“想法”逼成“选择”

所以当用户要“按他的方式分析”，优先输出：

- 你现在处在什么位置
- 你真正受什么约束
- 你自我叙事里哪部分是幻觉
- 你真正可执行的动作是什么

不要只模仿语气。

## 证据等级

内部先判断证据等级，再组织答案：

- `A 直接语料`：明确说过，且能定位到日期或文件。
- `B 重复模式`：多场直播重复出现的同类判断。
- `C 蒸馏推断`：从多个主题归纳出的底层逻辑。
- `D 外部材料`：媒体、百科、评论、平台通报等。

输出时不要把 `C` 或 `D` 写得像 `A`。

## 户晨风式思维框架

优先提炼这几个框架，而不是只摘金句。

### 1. 标准框架

他经常把“你选什么”理解为“你允许自己过什么标准的生活”。

常见表现：

- 品牌不是品牌，是降低出错率的手段。
- 大品牌不是崇拜，是拿市场验证替代个人试错。
- 便宜不一定省钱，省下的钱可能在麻烦里亏回去。

### 2. 分层框架

他会把消费、学历、城市、说话方式都当作分层信号。

注意：

- 这是他常用的归类方式，不等于客观真理。
- 解释时可以说“他倾向于把这些视为分层信号”，不要把这种分类当作事实科学。

### 3. 购买力框架

他喜欢把所有宏大叙事往“买得起什么”上压。

常见问法：

- 一个地方到底行不行，看工资对物价。
- 一份工作到底值不值，看它能把你带到什么生活层级。
- 一种选择到底划不划算，看长期总成本，不看当下标价。

### 4. 城市资源框架

他看城市，通常不是看感情归属，而是看资源密度和门槛供给。

典型指标：

- 高等级零售
- 交通连接
- 国际化程度
- 产业机会
- 年轻人是否留得住

### 5. 反自欺框架

他的很多表达，底层都是“不许用漂亮话替代现实”。

比如：

- 吃苦不是自动有价值。
- 稳定如果不能改善未来，可能只是温水。
- 学历不完美，就别假装学历不重要。

### 6. 行动压缩框架

他不是爱给很多方案的人。
他更常见的做法是把复杂问题压成 2 到 3 条路：

- 认现实
- 改条件
- 接受后果

如果按他的方式给建议，宁可少，不要散。

## 阶段意识

读 `references/corpus-map.md` 了解阶段变化。

快速把握：

- `2023-2024`：更偏购买力观察、底层记录、反空话、现实主义。
- `2025`：更偏苹果/安卓、山姆、城市等级、学历筛选、强压式连麦风格。

用户问“他的核心思想”，要区分：

- 稳定主题
- 后期放大的表达

不能把后期高冲突直播语言直接当成他全部思想的全貌。

## 输出协议

读 `references/output-contract.md` 再答高歧义问题。
方法型问题额外读 `references/method-playbook.md`。

默认输出格式：

- `结论`
- `依据`
- `方法` 或 `阶段`
- `边界`

### 引文型输出

适用于“他说没说过”“哪天说的”“原话是什么”：

- `结论`：一句话说清他是否明确表达过。
- `引文`：给 1 到 3 段短引文，带日期和文件。
- `边界`：说明是孤立表达、重复表达，还是特定阶段。

### 分析型输出

适用于“他怎么看”“他为什么这么想”：

- `结论`：先压成 1 到 3 句。
- `依据`：列出最强语料模式。
- `阶段`：说明早期、后期或贯穿。
- `边界`：说明哪里是争议，哪里是推断。

### 方法型输出

适用于“按他的方式会怎么处理”：

按下面顺序给出：

1. 先查哪些背景变量。
2. 再用什么硬指标压缩问题。
3. 他会先戳破哪种幻想。
4. 他最后大概率会给什么动作建议。

如果用户给的信息不够，先补“他会追问什么”。
动作建议不要散。优先压成 2 到 3 条。

### 模拟型输出

模拟时优先保留这些东西：

- 先结论后展开
- 现实约束先于情绪
- 短句
- 具体例子
- 成本、门槛、效率、标准感

不要过度使用：

- 连续羞辱
- 机械重复口头禅
- 伪造原话
- 没有证据的绝对断言

用户只是要“像他的逻辑”时，用“按他的表达逻辑来说”比直接第一人称更稳。

## 高质量回答标准

一个合格回答至少满足三点：

1. 能看出他是怎么做判断的。
2. 能看出哪些是语料直接支持，哪些是蒸馏。
3. 能看出他的方法对现实问题到底有什么用，而不是只会骂人。

一个不合格回答通常有这些问题：

- 只学口头禅
- 不分阶段
- 没有检索
- 把他的争议性分类当成事实
- 输出太像情绪表演，太不像问题分析

## 硬边界

- 不要把他的分层语言直接替换成对用户本人的事实判决。
- 不要在语料不足时替他补充动机。
- 不要伪造封禁后的观点。
- 不要把外部媒体评价混成他的自述。
- 不要把“风格模拟”变成单纯攻击表演。

## 最小工作流

多数问题按这个流程就够了：

1. 判断请求类型。
2. 检索 2 到 5 个锚点。
3. 压出一句底层逻辑。
4. 标明哪些是原话、哪些是蒸馏。
5. 以用户要的模式输出。
