# Topic Index

这个文件用于两种场景：

- 用户问题很泛，需要先判断该往哪一组语料里搜。
- 用户问题很具体，但想提高检索命中率，需要一组更稳的锚词。

先选主题，再选 2 到 4 个锚词检索。

## 1. 苹果 / 安卓 / 标准感

适用问题：

- 他为什么总讲苹果和安卓
- 苹果人、安卓人到底是什么意思
- 他为什么把手机上升成生活标准

优先锚词：

- `苹果`
- `安卓`
- `iPhone`
- `标准`
- `系统`
- `残值`
- `能用`

补充锚词：

- `小米`
- `华为`
- `性价比`
- `高端`
- `低端`

重点看：

- 他如何把设备选择迁移成生活标准判断。
- 他如何把“总持有成本”替代“首发价格”。
- 他如何把手机变成分层信号。

阶段提示：

- 2025 年更强，更成体系。

## 10. 争议 / 矛盾 / 阶段变化

适用问题：

- 他前后有没有变

- `购买力`
- `底层`
- `苹果`
- `安卓`
- `学历`
- `城市`

检索建议：

- 同一主题至少查早期一次、后期一次。
- 优先对比 2023-2024 与 2025。

重点看：

- 早期现实记录
- 后期强分类表达
- 同情底层与标签化底层之间的张力

## 快速检索模板

### 用户问“他怎么看某个东西”

先搜：

```powershell
python scripts/search_corpus.py "<主题词1> <主题词2>" --max-hits 6
```

### 用户问“他是不是一直这么想”

分别搜：

```powershell
python scripts/search_corpus.py "<主题词>" --to-date 2024-12-31 --max-hits 5
python scripts/search_corpus.py "<主题词>" --from-date 2025-01-01 --max-hits 5
```

### 用户问“他的方法是什么”

优先从这些主题交叉搜：

- `学历 收入 城市 手机`
- `购买力 标准 选择`
- `直接说 快点 别废话`
